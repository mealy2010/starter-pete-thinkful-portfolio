# Pete Thinkful Portfolio Redesign

## Project Description

This project is a redesigned version of Pete Thinkful’s original portfolio website. The goal of the redesign was to improve the overall visual layout, strengthen the structure of the page, and create a more polished portfolio experience while still keeping the site simple, readable, and easy to navigate. The updated design focuses on better spacing, cleaner alignment, stronger visual hierarchy, and a more intentional presentation of Pete’s artwork and background.

The redesigned site includes a refreshed About section, a more organized Portfolio section, and a Contact section that is easier to locate and use. The redesign also maintains semantic HTML and clean CSS styling so that the site remains accessible, structured, and aligned with project expectations.

---

## Original and Revised Layout

### Original Layout
The original portfolio site had a basic structure with the required sections, but the layout was more plain and less visually organized. Content was present, but spacing, alignment, and image presentation were minimal. The site functioned correctly, but it did not feel as cohesive or polished as a finished portfolio page.

### Revised Layout
The revised layout keeps the same overall sections—About, Portfolio, and Contact—but improves the page by introducing a cleaner design system. The About section was redesigned into a more balanced layout with a larger circular image and aligned text content. The Portfolio section was adjusted to present the artwork more clearly and consistently. Typography, spacing, image sizing, and alignment were revised to make the page look more intentional and professional.

## Redesign Mockup

### Theme
The redesign uses a clean artistic portfolio theme that feels simple, warm, and professional. The design supports Pete’s identity as an artist by using a soft background color, centered section headings, circular portrait styling, and consistent image presentation in the portfolio section.

The redesigned page includes the following components:

## Implementation Plan

The implementation plan for the redesign followed these steps:

1. Review the original portfolio layout and identify areas needing visual improvement.
2. Plan the updated theme, spacing, alignment, and content hierarchy.
3. Update the HTML structure where needed while keeping semantic elements such as `header`, `nav`, `section`, `article`, and `footer`.
4. Revise the CSS to improve layout, typography, image styling, spacing, and overall consistency.
5. Test the design in the browser and adjust alignment, image sizing, and spacing until the page matched the redesign goals.
6. Publish the updated project through GitHub Pages and verify that all images and links worked correctly.

## Design Trade-Offs

Several design trade-offs were made during this redesign:

- A simple layout was chosen over a more complex design so the page would remain easy to read and meet assignment requirements.
- Flexbox was used for alignment and spacing because it provided cleaner layout control without overcomplicating the CSS.
- The color palette was kept minimal to maintain a professional look and avoid distracting from the portfolio images.
- Portfolio descriptions were simplified visually so the artwork remained the focal point.
- The redesign balanced creativity with structure to ensure the page still functioned as a clear portfolio site rather than just a decorative design.

## Key Decisions

Several key decisions shaped the redesign:

- Keeping the page to three clear sections made the portfolio easier to navigate.
- Enlarging and styling Pete’s profile image as a circle created a stronger focal point in the About section.
- Centering major headings improved the visual hierarchy.
- Using consistent image sizes in the Portfolio section created a cleaner and more professional appearance.
- Choosing a limited color palette helped the page feel cohesive and intentional.

---

## Challenges, Debugging, and Troubleshooting

A major challenge during development involved getting all portfolio images to display correctly on the live site. Some images appeared in the local project but not on GitHub Pages. This required checking file names, folder placement, image paths, and GitHub Pages publishing behavior. The issue was resolved by confirming exact file names, correcting HTML image references, and allowing time for GitHub Pages to refresh after updates.

Another challenge involved aligning the About section so the image and text looked balanced. This required revising CSS multiple times, adjusting image size, spacing, and layout structure until the section felt visually even.

There were also troubleshooting moments related to page deployment and viewing the live site. Learning how to properly enable GitHub Pages and test the deployed version was an important part of the project.
